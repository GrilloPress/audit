The health and care system is complicated. And the use and processing of data is varied.

Users should not have to understand how data and records are created and managed to:

  • take ownership, in part, of their health records
  • see what information the NHS has about them

However, if a user wishes to take ownership or see that information, they should be set up for success.

The system should provide users with plain English explanations of what data the NHS has, how it is used, why it is shared and with whom, when the user wishes to see that information more clearly.

If there are no clear reasons for processsing or keeping data, the NHS should not be processsing or keeping that data.

The system should not try to explain the "universe", but provide just enough information, just in time to satisfy users' needs.

Before a user sees complex medical records or audit data, the system should:

  • clearly state what data the NHS has
  • why that data is used for their care or beyond their care

This information will likely be complicated, or introduce new concepts or modes of data processing. This information should be:

  • in plain language
  • explained bit by bit in a journey
  • give context to data they can see or manage

Examples

Explainer cards

  • Series of cards users see before they see a timeline of their Summary Care Record use

Hypotheses

If we use plain language to explain step by step, how complex systems (like Summary Care Record) work
then users will know what information they are seeing and how it is used

We know this works when:

  • users can state why the NHS uses the data they are seeing
  • users are able to easily navigate data when they see it
  • users report the data from their health records is in a format "as expected" and intuitive
